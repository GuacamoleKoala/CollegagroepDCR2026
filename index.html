Workshop:
Wikidata naar Netwerk

Welkom bij de workshop voor de Collegagroep Digitale Collectieregistratie 2026. Vandaag leren we hoe we erfgoeddata (zoals die van Guido Gezelle) uit Wikidata omzetten in betekenisvolle netwerkvisualisaties. We doorlopen samen de 4 stappen uit de presentatie.

Slides Presentatie Workshop Repository Gezelle Leiddraad
01

Wikidata Bevragen met SPARQL

30 min

We starten bij de bron. We gebruiken de Wikidata Query Service om Linked Open Data rondom Guido Gezelle (of je eigen collectie) op te halen. Je kunt hiervoor zelf queries schrijven of generatieve AI om hulp vragen.

# Voorbeeld: Relaties van Guido Gezelle
SELECT ?item ?itemLabel ?student ?studentLabel WHERE {
  ?item wdt:P106 wd:Q36180 .
  ?student wdt:P1066 ?item .
  SERVICE wikibase:label { bd:serviceParam wikibase:language "nl". }
} LIMIT 100

02

Data Opschonen

30 min

Netwerksoftware vereist een specifieke structuur. We splitsen onze ruwe data op in twee bestanden: Nodes (de knopen/personen) en Edges (de randen/relaties). Loop je vast? Gebruik dan de opgeschoonde voorbeeldbestanden hieronder om direct verder te kunnen.

Opgeschoonde bestanden (Inpikpunt):

nodes.csv Lijst van alle entiteiten/personen
edges.csv De verbindingen tussen de nodes
03

Visualiseren in Gephi

30 min

We importeren de Nodes en Edges in Gephi. We passen het ForceAtlas2 algoritme toe om het netwerk open te trekken en berekenen statistieken (zoals degree) om knopen de juiste grootte en kleur te geven.

Gephi Projectbestand (Inpikpunt):

GephiProject.gephi Volledig gestyled Gephi-project
04

Publiceren met Github

15 min

Zodra het netwerk goed staat, exporteren we dit via de Sigma Exporter plugin. Om de website daarna naar je hand te zetten (titel, legende, layout), passen we de bestanden in de Github repository aan. Via Github Pages zetten we het online!

Aan te passen bestanden in repo (/network):

config.json Pas hier titel, tekst en "maxEdgeSize" aan
index.html HTML structuur van je visualisatie
main.js Javascript en zoekfunctionaliteiten
